## 一、项目概述

中国地质大学（北京）校内学习资料分享平台。学生用学号登录，可浏览课程文件夹、在线预览和下载往年真题。管理员审核上传内容。

## 二、技术架构

### 整体
```
前端 (Vue 3 + Element Plus)  →  Nginx 反向代理  →  Node.js Express 后端  →  SQLite 数据库
    客户端渲染 SPA               静态文件 + API 代理        RESTful API                 文件型
```

### 后端
- **框架**：Express 4.x
- **数据库**：sql.js（WASM 版 SQLite，无需编译，跨平台兼容）
  - WAL 模式
  - 每次写入操作后调用 `saveDb()` 写回磁盘（sql.js 写入整个数据库文件）
- **JWT 认证**：access token（2h）+ refresh token（7d）
- **文件上传**：Multer + UUID 文件名，存到 `uploads/` 目录
- **上传限制**：最多10人同时上传
- **下载限速**：每线程600KB/s，Transform Stream实现
- **Office预览**：LibreOffice 7.3 转换 doc/docx/ppt/pptx/xls/xlsx 为 PDF
- **登录限流**：每IP错误5次锁定30分钟，内存存储
- **访问统计**：60秒内同IP去重

### 前端
- **Vue 3** + Vite 6
- **UI 框架**：Element Plus（中文语言包）
- **状态管理**：Pinia
- **请求库**：Axios（带 JWT 拦截器 + 自动刷新 token）
- **图表**：ECharts（管理后台仪表盘）

## 六、核心功能实现

### 6.1 学生登录
- 学号验证规则：10 位数字，前四位 1000-1030，后两位 00-36
- 首次登录自动创建账号（无密码）
- 错误信息统一返回「学号错误」，不暴露格式规则

### 6.2 课程文件夹浏览
- 首页显示课程卡片，每个课程名 + 资源数量
- 搜索课程支持**模糊匹配**（按字序匹配，如"马原"匹配"马克思主义基本原理"）
- 点击课程进入该课程的资源列表

### 6.3 资源搜索（课程内）
- 支持模糊匹配（同课程搜索逻辑）
- 可筛选资料类型（考试真题 / 复习资料 / 其他资料）
- 分页，默认每页 24 条

### 6.4 上传与审核
- 学生上传 → 状态为 pending
- 管理员在后台审核 → 通过/驳回
- 审核通过后文件对所有学生可见
- 上传时「所属课程」为选填

### 6.5 在线预览
- **PDF**：`Content-Disposition: inline` 直接在浏览器打开
- **Office 文件**（doc/docx/ppt/pptx/xls/xlsx）：服务器调用 LibreOffice 转换为 PDF
  - 转换后的 PDF 缓存到 `uploads/preview_cache/` 目录
  - 已安装文泉驿正黑 + Noto Sans CJK 中文字体
- 预览不需要等下载完成，实时渲染

### 6.6 下载限速
- 每线程 600KB/s
- 使用 Transform Stream 实现节流
- 前 100KB 无延迟（保证响应速度），之后平滑限速
- 前端使用 `window.open()` 直接打开下载链接，浏览器原生下载框立刻弹出

### 6.7 手机端适配
- 登录页自适应内边距
- 课程卡片手机端 2 列显示
- 管理后台侧边栏手机端自动隐藏，点击左上角滑入
- 表格超出时横向滚动
- 自动检测移动端，管理后台跳转到手机版 `/admin/mobile`

### 6.8 访问统计
- 每次 API 请求记录 IP + User-Agent 到 `visit_logs` 表
- 同 IP **60 秒内去重**，防止页面多 API 请求重复计数
- 仪表盘显示：今日访问人数（去重IP）、今日访问量（所有请求）、总访问量

### 6.9 上传并发限制
- 最多 **10 人同时上传**
- 超出时返回 429「当前上传队列已满」

### 6.10 资源管理（管理员）
- 资源列表可直接**编辑标题、移动课程、修改分类**
- 审核弹窗支持预览文件后通过/驳回
- 支持直接在管理后台**上传资料**

## 八、Nginx 配置要点

- HTTP (80) 和 HTTPS (443) 都代理 `/api/` 到 `127.0.0.1:3000`
- HTTP 默认 `return 301 https://`
- 静态文件直接由 Nginx 从 `/opt/resource-platform/client/dist` 提供
- `client_max_body_size 60M`（上传文件限制）
- `gzip on` 开启压缩
- `proxy_set_header X-Real-IP $remote_addr`（用于正确获取客户端 IP）

### 当前状态
由于域名未备案，DNS 已删除，目前仅通过 `http://39.105.108.171` 访问。Nginx 配置中 `server_name _` 处理 IP 访问。

## 九、注意事项

### 9.1 学号验证规则（安全）
- 规则不显示在 UI 上，错误统一返回「学号错误」
- 规则仅同学口口相传：10 位数字，前四位 1000-1030，后两位 00-36

### 9.2 文件名乱码问题
- 通过批量上传脚本（手动拼 multipart form-data）上传的文件，`file_name` 字段在数据库中已乱码
- **下载时使用 `title` 字段作为文件名**，而非 `file_name`
- 预览时也使用 `title` 字段

### 9.3 登录限流
- 5 次错误锁定 30 分钟（内存存储，重启服务即重置）
- 区分 IP，不同 IP 互不影响
- 需要在 Express 中设置 `app.set('trust proxy', 1)`（因 Nginx 反代）

### 9.4 批量上传
- 脚本位于 `server/seeds/batch-upload.js`
- 本地运行，通过 HTTP 上传到服务器
- 按文件名自动判断分类（包含"期末""期中""试题"等关键词的归为「考试真题」）
- 有子目录结构的按子目录名判断（真题 / 复习资料）

### 9.5 下载限速
- 每连接 600KB/s
- 使用 Transform Stream 实现
- 前端用 `window.open` + token 参数方式，确保浏览器立即弹出下载框

### 9.6 访问统计去重
- 同一 IP 60 秒内多次 API 请求只计 1 次访问
- 用内存 Map 存储最近访问时间

### 9.7 上传并发限制
- 最多 10 人同时上传
- 内存计数器，服务重启后重置

### 9.6 Office 预览
- 依赖 LibreOffice，需安装 fonts-wqy-zenhei + fonts-noto-cjk 中文字体
- 转换速度取决于文件大小，首次转换有延迟
- 缓存到 `preview_cache/`，同名文件不重复转换

### 9.7 服务器内存
- 日常占用约 500MB
- Node.js V8 堆限制 256MB（`--max-old-space-size=256`）
- systemd 自动重启保活

## 十二、关键对话记录

### 功能变更
1. **贝壳 → 北地书阁**：原名"贝壳真题库"因北科（北京科技大学）的昵称就是"贝壳"，怕混淆，改为用户命名的"北地书阁"
2. **学生登录**：从用户名密码改为纯学号验证（10位，前四位1000-1030，后两位00-36），规则不展示在UI上
3. **资料类型**从5类（课件/试卷/实验报告/参考资料/其他）改为3类（考试真题/复习资料/其他资料）
4. **课程文件夹**：首页改为课程卡片展示，点击进入资源列表，支持模糊搜索
5. **下载限速**：从无限制 → 5MB/s → 200KB/s → 600KB/s
6. **在线预览**：PDF inline + Office 通过 LibreOffice 转 PDF
7. **登录限流**：5次错误锁定30分钟

### 重要问题
1. **文件名乱码**：批量上传脚本手动构造 multipart 导致 file_name 乱码，解决方案：下载时使用 title 字段
2. **download API 500**：移除 auth 中间件后未设置 `req.user`，导致 `req.user.id` 报错，已修复
3. **下载响应慢**：前端用 `fetch()` + `res.blob()` 需等整个文件下载完才弹出保存框，改为 `window.open()` 立即弹出
4. **Office 预览乱码**：服务器缺少中文字体，安装 fonts-wqy-zenhei + fonts-noto-cjk 后解决
5. **登录限流计数不增加**：使用 Map 对象时计数不累加，改用普通对象后正常
